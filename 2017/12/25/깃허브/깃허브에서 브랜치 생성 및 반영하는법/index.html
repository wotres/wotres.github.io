깃허브에서 브랜치 생성 및 반영하는법

  • 명령순서
    git branch 브랜치명 // 브랜치명 생성
    git branch // 현재 생성된 브랜치와 활성화된 브랜치를 확인한다.
    git checkout 원하는브랜치명 // 원하는 브랜치 명으로 활성화
    해당폴더에 파일 ex) python 파일 생성
    git add -A // 깃에 모든 파일 반영
    git commit -m “설명” // 반영된 파일 설명 및 깃 반영
    git push origin 브랜치명 // 브랜치에서 반영된 정보를 깃에 반영한다.
    깃허브 홈페이지에서 해당 브랜치명으로파일 올라온것 확인후 Compare & pull request 버튼 클릭
    깃 머지 실행
    git checkout master // master로 이동
    git pull // pull 명령어로 로컬저장소를 원격 저장소와 같게 만든다. 기준은 원격저장소
    위 pull에서 조심해야하는게 작업해놓은게 있는데 커밋하지않고 마스터로 이동한뒤 pull 땡기면 파일 사라짐
    git merge 브랜치명 // 병합작업 수행
    이부분에서 오류 잘남 why? 서로 다른 깃내려받은뒤 작업 및 반영 시점에 따라 다를 수 있기 때문
    충돌날 경우 파일 수정 작업 수행
    git status // 상태확인 후
    git add -A // 반영
    git push origin master // 마스터를 push 하여 원격 저장소에 업로드함\
    git branch -d 브랜치명 // 해당 브랜치 삭제
