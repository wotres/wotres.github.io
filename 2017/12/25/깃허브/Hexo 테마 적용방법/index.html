hexo 테마 적용하는법

보기좋은 화면이 활용하기도 편리하다. 따라서 테마(보이는 방식과 화면)을 변경한다.
https://hexo.io/themes/
위 사이트에서 다양한 테마 중 마음에 드는것을 고른뒤 깃허브에 들어가 다운로드 한 뒤 zip을 풀어 내 폴더에 설치한다.
ex)blog/themes/ 아래에 설치

개인적으로는 깔끔한걸 선호하여 Clean-blog라는 것을 찾아서 설치
https://github.com/klugjo/hexo-theme-clean-blog.git
에서 clone or download를 클릭하여 직접 다운로드 혹은
cmd 창의 blog/themes/ 해당 경로 아래에서
git clone https://github.com/klugjo/hexo-theme-clean-blog.git

그럼 내부에 hexo-theme-clean-blog 라는 폴더 설치됨
이름 clean-blog 로 수정

가끔 _config.yml.sample 라고 된 파일이 있는데 _config.yml로 변경하여 사용하면 된다.

해당테마의 설정을 변경하는 _config.yml 파일을 수정해본다.

태그등을 이용하여 문서나 데이터의 구조를 명기하는 언어

블로그 경로의 _config.yml 은 hexo 블로그 전반적인 설정이며
themes/테마 폴더의 경로의 _config.yml 은 hexo 블로그에 적용된 테마의 설정

페이지 만들기

Tags 페이지 생성
tag 페이지를 만들어 관리한다. post 만들때와 비슷

  • 명령어
    hexo new page “tags”

Categories 페이지 생성

Categories 페이지를 만들어 관리

  • 명령어
    hexo new page “categories”

포스트 쓰기

이제 글을 쓸 수 있음
hexo new post “포스트 이름”

title: “” // post 제목
date: 2017-12-25 00:00:00 // post 생성 날짜
tags: [“”] // “” 안에 태그 입력
cover: // post 커버 이미지

수정한 파일들 반영

  • 명령어
    hexo clean 을 통해 deploy 전에 정리한다.
    hexo generate 를 이용하면 github에 올라갈 리소스만 정리되어 따로 폴더가 생성된다.
    hexo deploy 를 통해 배포한다.
    보통은 편하게 아래와 같이 입력함
    hexo deploy –generate
    hexo d g 라고도 사용해도 됨…

기타 명령어

처음 생성된 hellowworld 파일 삭제함

rm source/_posts/hello

이미지 입력

아래와 같은 명령어 입력

글쓰기

hexo new ‘hello world’

3가지 레이아웃

post : 일반 포스트를 저장하는 폴더(기본값은 post)
page : About같은 해당 경로로 접근해야 볼 수 있는 페이지.
draft : 바로 게시하지 않고 작성할 수 있는 초안입.

기타 정보들

layout 레이아웃
title 제목
date 배포한 날짜 파일 생성 날짜
updated 수정된 날짜 파일 생성 날짜
comments 코멘트 기능 여부 true
tags 태그 (Page 에서는 사용 불가)
categories 카테고리 (Page 에서는 사용 불가)
permalink 포스트의 URL 을 수동으로 설정 가능
thumbnail 썸네일 지정 본문 첫번째 이미지

Asset Folders

source 안에 있는 파일들이 올라감

Global Asset Folders

/img/이미지이름.jpg 와 같은 식으로 상대주소 적어서 사용가능

post_asset_folder: true

true 로 설정 후
hexo new [layout] 명령어로 새로운 포스트를 생성할 때마다 새 폴더를 생성

hexo generate

하면 아래 public 이란 것이 생성되고 올라갈 준비함
